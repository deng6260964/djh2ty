# Browser-use 集成测试用例：老师工作台

> 状态：当前
> 范围：管理端
> 更新：2026-04-26
> 目标页面：`http://localhost:3000/dashboard`
> 覆盖范围：老师端 V2 工作台首屏、登录跳转、关键提醒、快速排课入口、快速操作导航。
> 执行方式：使用 Codex in-app browser 的 `browser-use` 技能执行，优先使用 DOM snapshot 中可见的 role/name/text 定位。

## 前置条件

- 后端服务运行在 `http://localhost:8000`
- 前端服务运行在 `http://localhost:3000`
- 数据库已初始化默认账号：`admin / admin123`
- 浏览器可访问 `http://localhost:3000/dashboard`

## 定位约定

- 页面断言优先使用 `domSnapshot()` 中的角色、文本和按钮名称。
- 点击前先确认 locator 数量，若数量不唯一，缩小作用域或使用当前快照里的更稳定文本。
- 对于重复按钮，例如顶部和底部都存在的“快速排课”，允许先通过 DOM 快照确认重复数量，再按可见顺序选择目标入口。

## 用例 BU-DASH-001：未登录访问工作台会跳转登录页

步骤：

1. 清理本地登录态，访问 `/dashboard`。
2. 等待页面加载。

预期：

- 当前 URL 为 `/login`。
- 页面存在标题“家教辅助系统”。
- 页面存在用户名输入框、密码输入框和“登 录”按钮。

## 用例 BU-DASH-002：默认管理员登录后进入工作台

步骤：

1. 在登录页输入用户名 `admin`。
2. 输入密码 `admin123`。
3. 点击“登 录”。

预期：

- 当前页面进入 `/dashboard`。
- 顶部用户区域显示“管理员”。
- 左侧导航选中或展示“工作台”。
- 主内容存在标题“工作台”。

## 用例 BU-DASH-003：工作台首屏符合 V2 信息结构

步骤：

1. 在已登录状态访问 `/dashboard`。
2. 获取 DOM 快照。

预期：

- 标题区存在“工作台”和“今天是”。
- 操作区存在“今天”和“快速排课”按钮。
- 存在“关键提醒”区。
- 关键提醒包含：`待补记录`、`今日课程`、`待收费`、`待批改`。
- 工作区包含：`今日课程`、`待补记录`、`待收费提醒`、`待批改作业`。
- 底部存在“快速操作”区。
- 快速操作包含：`快速排课`、`快速调课`、`新增学生`、`录入收款`。

## 用例 BU-DASH-004：空数据状态可读且不破坏布局

步骤：

1. 使用新初始化数据库或无业务数据账号访问工作台。
2. 获取 DOM 快照。

预期：

- 今日课程区显示“今天暂无课程”。
- 待补记录区显示“没有待补记录课程”。
- 待收费提醒区显示“暂无待收费学生”。
- 待批改作业区显示“暂无待批改作业”。
- 页面仍保留关键提醒和快速操作入口。

## 用例 BU-DASH-005：顶部“快速排课”打开排课弹窗

步骤：

1. 在工作台点击顶部“快速排课”按钮。
2. 获取 DOM 快照。

预期：

- 出现标题为“快速排课”的弹窗。
- 弹窗包含字段：学生、科目、上课日期、开始时间、结束时间、上课地点、备注。
- 上课日期默认填入当天日期。
- 弹窗存在“取消”和“保存课程”按钮。

## 用例 BU-DASH-006：快速操作“快速调课”跳转课程页

步骤：

1. 在工作台点击“快速调课”。
2. 等待页面加载。

预期：

- 当前 URL 为 `/courses`。
- 页面存在课程页标题或周视图/列表视图切换入口。
