# Browser-use 全量集成测试用例

> 状态：当前
> 范围：管理端
> 更新：2026-04-26
> 目标：覆盖老师端 V2 当前已实现的主要功能模块、关键表单、导航入口和跨模块业务闭环。
> 执行工具：Codex in-app browser + `browser-use` 技能。
> 适用环境：本地开发环境 `http://localhost:3000` + `http://localhost:8000`。

## 1. 执行约定

### 1.1 前置条件

- 后端 FastAPI 已启动：`http://localhost:8000`
- 前端 Vite 已启动：`http://localhost:3000`
- 数据库可用，并已初始化默认管理员：`admin / admin123`
- 浏览器优先从无登录态开始验证认证与路由守卫
- 如执行破坏性用例，建议使用临时测试数据，例如学生名以 `BU测试` 开头

### 1.2 browser-use 定位规则

- 每次点击、输入、选择前先获取或复用当前 DOM snapshot。
- 优先使用 `role/name`、placeholder、弹窗标题、按钮文本定位。
- 对重复按钮先确认数量，再通过滚动或当前可见区域选择目标。
- 表格行操作先通过学生名、课程科目、作业标题等业务文本确认目标行。
- 文件上传、删除、提交类动作执行前确认测试数据范围。

### 1.3 推荐测试数据

| 类型 | 建议值 |
| --- | --- |
| 学生姓名 | `BU测试学生A` |
| 年级 | `初二` |
| 科目 | `数学`、`英语` |
| 家长姓名 | `BU家长A` |
| 家长电话 | `18800000001` |
| 课程时间 | 明天 09:00-10:30 |
| 作业标题 | `BU数学练习-一次函数` |
| 反馈内容 | `BU课堂表现积极，需加强计算准确率` |
| 资料标题 | `BU测试资料` |
| 充值金额 | `300` |

## 12. 跨模块核心闭环

### BU-FLOW-001 学生 -> 充值 -> 排课 -> 完成课程 -> 余额扣减

步骤：

1. 新增测试学生。
2. 在收费页给该学生记录预收充值。
3. 在课程页给该学生排一节课程。
4. 完成该课程。
5. 回到学生详情或收费页查看账户。

预期：

- 学生创建成功。
- 充值后余额增加。
- 排课后课程显示待上课。
- 完成课程后状态变为已完成。
- 账户余额按课时单价扣减。

### BU-FLOW-002 余额不足 -> 工作台/收费提醒出现

步骤：

1. 创建或选择余额不足的学生。
2. 为其排下一节预计费用高于余额的课程。
3. 访问工作台和收费页。

预期：

- 工作台“待收费”数量增加。
- 待收费提醒中出现该学生。
- 收费页“待收费学生”显示预计缺口。

### BU-FLOW-003 作业布置 -> 待批改提醒

步骤：

1. 为测试学生布置作业。
2. 将作业置为待批改状态，或使用已有已提交作业。
3. 访问工作台。

预期：

- 工作台“待批改”数量反映该作业。
- 作业中心“待批改”标签能看到该作业。

### BU-FLOW-004 课程反馈 -> 反馈复盘

步骤：

1. 为已完成课程创建反馈。
2. 访问反馈复盘页。
3. 查看反馈详情。

预期：

- 反馈复盘列表出现该反馈。
- 详情内容与创建时一致。
- 未推送反馈可执行推送。

### BU-FLOW-005 资料上传 -> 分享给学生 -> 学生详情可追踪

步骤：

1. 上传测试资料。
2. 分享给测试学生。
3. 查看资料卡片和学生详情相关信息。

预期：

- 资料显示已分享给测试学生。
- 分享操作无报错。

## 14. 删除功能专项回归

> 说明：删除动作会改变本地数据。执行以下用例前，必须确认目标数据均为本轮测试创建的数据，推荐统一使用 `BU测试` 前缀。执行时如由 Codex 操作浏览器，点击最终确认删除前需要用户动作确认。

### BU-DEL-001 删除课程

前置：

- 已存在一条测试课程，学生名或备注包含 `BU测试`。
- 课程状态不限，但建议使用单独创建的待上课测试课程。

步骤：

1. 访问 `/courses`。
2. 切换到“列表”视图，或在周视图中定位测试课程卡片。
3. 确认目标行/卡片包含测试学生姓名、科目和时间。
4. 点击该课程的“删除”。
5. 在确认框中确认文案为“确认删除此课程？”。
6. 点击“删除”确认。
7. 刷新当前课程页或重新筛选。

预期：

- 出现“课程已删除”。
- 目标课程不再出现在周视图或列表视图。
- 工作台今日课程数量随之减少或不再显示该课程。

### BU-DEL-002 删除学生

前置：

- 已存在测试学生，姓名以 `BU测试` 开头。
- 如需避免级联/关联影响，优先删除无课程、无作业、无资料分享的测试学生。

步骤：

1. 访问 `/students`。
2. 搜索测试学生姓名。
3. 确认表格行中姓名、年级、家长电话均为测试数据。
4. 点击该行“删除”。
5. 在确认框确认文案包含学生姓名。
6. 点击“确认删除”。
7. 重新搜索该学生姓名。

预期：

- 出现“已删除学生 学生名”。
- 搜索结果不再显示该学生。
- 相关列表/选择器中不再出现该学生。

### BU-DEL-003 删除作业

前置：

- 已存在测试作业，标题以 `BU` 或 `BU测试` 开头。

步骤：

1. 访问 `/assignments`。
2. 在“全部”或对应状态标签中定位测试作业。
3. 确认作业标题、科目、截止日期为测试数据。
4. 点击删除按钮。
5. 在确认框确认文案为“确认删除此作业？”。
6. 点击“删除”。
7. 刷新作业列表或重新进入标签页。

预期：

- 出现“作业已删除”。
- 目标作业不再出现在全部/状态标签页。
- 工作台待批改/待提交数量不再包含该作业。

### BU-DEL-004 删除课堂反馈

前置：

- 已存在测试反馈，反馈内容包含 `BU` 或关联测试课程/测试学生。

步骤：

1. 访问 `/feedback`。
2. 通过学生筛选或表格内容定位测试反馈。
3. 可先打开“课堂反馈详情”，确认内容为测试数据。
4. 关闭详情后点击删除按钮。
5. 在确认框确认文案为“确认删除此反馈？”。
6. 点击“删除”。
7. 刷新反馈列表。

预期：

- 出现“反馈已删除”。
- 目标反馈不再出现在反馈列表。
- 本页反馈数减少。

### BU-DEL-005 删除成绩记录

前置：

- 已存在测试成绩记录，考试名称包含 `BU`。
- `/progress` 页面可正常打开。

步骤：

1. 访问 `/progress`。
2. 选择测试学生。
3. 在“成绩记录”标签定位测试成绩。
4. 确认考试名称、科目、分数、考试日期为测试数据。
5. 点击该行删除按钮。
6. 如出现确认框，确认删除。
7. 刷新成绩记录。

预期：

- 出现“成绩记录已删除”。
- 目标成绩从表格移除。
- 已记录成绩数量和趋势图同步更新。

### BU-DEL-006 删除资料

前置：

- 已存在测试资料，标题包含 `BU测试资料`。
- 如资料已分享给学生，确认删除该资料不会影响非测试数据。

步骤：

1. 访问 `/resources`。
2. 在资料列表中定位测试资料卡片。
3. 确认标题、科目、类型、上传时间为测试数据。
4. 点击删除按钮。
5. 在确认框确认文案为“确认删除此资料？”。
6. 点击“删除”。
7. 刷新资料页。

预期：

- 出现“资料已删除”。
- 目标资料卡片不再显示。
- 若原列表只包含该资料，页面回到“还没有上传资料”空状态。

### BU-DEL-007 删除收费记录

前置：

- 已存在测试收费/充值记录，备注包含 `browser-use测试` 或 `BU测试`。
- 当前页面实现若没有暴露删除入口，则记录为“功能未实现/不可测”。

步骤：

1. 访问 `/billing`。
2. 切换到“收支记录”标签。
3. 定位测试收费或充值记录。
4. 确认学生、金额、收款方式、备注为测试数据。
5. 如页面存在删除按钮，点击删除。
6. 在确认框确认删除。
7. 刷新收支记录。

预期：

- 若有删除入口：记录被删除，统计同步更新。
- 若无删除入口：记录为“页面未提供删除功能”，用例结果标记为 `BLOCKED`，并记录缺失入口。

### BU-DEL-008 删除后跨页面一致性

前置：

- 已执行至少一个删除用例。

步骤：

1. 回到工作台 `/dashboard`。
2. 访问相关模块页面，例如删除课程后访问课程页和工作台；删除资料后访问资料页。
3. 刷新页面。

预期：

- 被删除数据不再从其他入口出现。
- 统计数字与列表状态一致。
- 页面不出现 404、500、白屏或 stale 数据。
