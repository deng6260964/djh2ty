# 新增进度同步文档目录

> 状态：当前
> 范围：文档、全项目
> 更新：2026-04-28

## 背景

当前文档结构已经将产品、设计、实现、测试、AI 工作日志和变更记录分层，但多人并行开发时仍缺少一个专门记录“进行态”的位置。实施计划适合描述怎么做，变更记录适合描述已经发生的事实，两者都不适合承载日常负责人同步和阻塞状态。

## 变更内容

- 新增 `docs/08-progress/` 作为项目级进度同步中心。
- 新增 `docs/08-progress/project-status.md` 记录当前工作流总览。
- 新增 `docs/08-progress/workstreams/` 存放并行工作流进度。
- 新增学生端新版承接、管理端回归刷新两个工作流示例。
- 更新文档入口、文档流程和文档地图，使进度同步成为推荐文档结构的一部分。

## 影响范围

- 文档检索顺序新增进行态入口。
- 多人并行开发时，计划和当前进度不再全部堆入 `docs/04-implementation/teacher-v2/plan.md`。
- 完成态变更仍需沉淀到 `docs/07-changes/`，不能只停留在 `docs/08-progress/`。

## 兼容性与风险

无代码兼容性影响。主要风险是进度文档长期不更新，因此要求工作流状态变化时同步更新 `docs/08-progress/project-status.md` 和对应工作流文档。

## 验证方式

- 检查新增文档均包含中文状态块。
- 检查 `docs/04-implementation/doc-map.md` 覆盖新增文档。
- 检查文档流程中明确 `04-implementation`、`07-changes` 与 `08-progress` 的边界。

## 后续事项

- 后续启动学生端新版或管理端回归刷新时，应优先更新对应工作流文档。
