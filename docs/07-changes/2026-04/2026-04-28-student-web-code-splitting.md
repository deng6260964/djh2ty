## 背景

学生端新版承接从待开始进入进行中。进度盘点时发现学生端仍是旧的移动端 Tab 结构，且 `npm run build` 产物集中在单个约 1.2 MB 的入口包，Vite 提示存在超过 500 kB 的 chunk。继续做新版页面前，需要先降低首包和后续迭代风险。

## 变更内容

- 将学生端课程、作业、反馈、进度、资料及详情页改为路由级懒加载。
- 为学生端 Vite 构建配置 React 和图表库共享 chunk。
- 保留现有学生端路由、页面文案和 API 行为，不改变业务数据或接口契约。

## 影响范围

- 产品文档：无产品行为变化。
- 设计文档：无页面结构或交互变化。
- 架构文档：无后端架构变化。
- 后端：无影响。
- 管理端：无影响。
- 学生端：影响 `student-web/src/App.tsx`、`student-web/vite.config.ts`。
- 微信小程序：无影响。

## 兼容性与风险

- 是否影响旧数据：否。
- 是否影响现有接口：否。
- 是否影响多人并行开发：低；变更集中在学生端入口和构建配置。
- 是否需要迁移或回滚方案：不需要迁移；如出现加载异常，可回滚路由懒加载和 Vite 手动拆包配置。

## 验证方式

- 命令：`cd student-web && npm run build`
- 页面：未做浏览器点击验证，本次为构建和加载边界改造。
- 数据：不涉及业务数据。
- 结果：构建通过，超 500 kB chunk 警告消失；仍存在既有 Vite CJS Node API deprecation warning。
