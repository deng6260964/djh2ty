## 一、通用交互规范

### 1.1 加载状态（Loading States）

**规则**：任何需要超过 300ms 的网络请求，都必须提供视觉反馈。

#### 首次进入页面（骨架屏）
适用场景：列表页、详情页首次加载。

```
骨架屏规范：
- 灰色占位块模拟真实内容的形状和大小
- 扫光动画（shimmer）：从左到右，1.5s 无限循环
- 颜色：基础 #E5E7EB，扫光 #F3F4F6
- 当数据加载完成时，骨架屏直接被真实内容替换（无动画过渡）
```

Web 端骨架屏示例：
```
+------------------------------------------+
|  [████████████ 150px]    [██████ 80px]  |  <- 页面标题骨架
|  ——————————————————————————————————————  |
|  [████ 50px]  [████████ 100px]  [██ 40px]  [████████ 80px]  |  <- 表头骨架
|  [████ 50px]  [████████ 100px]  [██ 40px]  [████████ 80px]  |  <- 行骨架
|  [████ 50px]  [████████ 100px]  [██ 40px]  [████████ 80px]  |
|  [████ 50px]  [████████ 100px]  [██ 40px]  [████████ 80px]  |
+------------------------------------------+
```

#### 局部刷新（Spinner）
适用场景：按钮提交、表格分页切换、筛选条件变更。

```
表现：
- 按钮提交中：按钮左侧显示旋转圈，文字变为"提交中..."，按钮变为禁用态
- 表格刷新：表格主体区域显示半透明蒙层 + 居中旋转圈
- 旋转圈颜色：#2563EB（主色）
- 旋转圈尺寸：按钮内 14px，表格蒙层 32px
```

#### 无限滚动加载（小程序端）
```
- 滚动到列表底部约 200rpx 时，自动触发加载
- 底部显示：[旋转圈] 加载更多...
- 全部加载完毕：底部显示"已显示全部内容"（灰色小字）
- 加载失败：底部显示"加载失败，点击重试"（可点击）
```

**规则**：列表无数据时，不显示空白页面，必须提供明确的视觉提示和引导操作。

#### 空状态组件结构

[插画图标]（灰色调，与页面功能相关）
[主提示文字]（14px，#374151）
[辅助说明]（13px，#9CA3AF，可选）

### 1.3 错误状态（Error States）

#### 网络错误
```
表现：
- 全页面错误：页面中央显示错误图标 + "网络异常，请检查网络后重试" + [重试] 按钮
- 局部错误（如表格加载失败）：在对应区域显示错误提示 + [重试] 链接
- Toast 错误（操作失败）：屏幕顶部显示红色 Toast，3秒后消失
```

#### 服务器错误（5xx）
```
- Toast 提示："服务器繁忙，请稍后重试"
- 保留用户输入内容，不清空表单
```

#### 404 页面
```
Web 端：
  居中显示：404 大字（灰色）+ "页面不存在" + [返回首页] 按钮

小程序端：
  显示通用错误空状态 + [返回上一页] 按钮
```

## 二、表单验证和错误提示规范

### 2.1 验证时机

| 触发时机         | 适用场景                                |
|------------------|-----------------------------------------|
| 实时验证（onChange）| 格式验证（手机号、日期格式）            |
| 失焦验证（onBlur）| 必填项、长度限制                        |
| 提交验证（onSubmit）| 整体校验，处理跨字段依赖（如结束时间必须晚于开始时间）|

**原则**：不在用户还在输入时就立刻报错，待失焦后再验证；但提交时对全部字段做完整校验。

### 2.2 错误展示位置

**输入框下方（推荐）**
```
[姓名 *]
[张____________]  <- 输入框，错误时边框变红
 请输入学生姓名   <- 错误文字，#EF4444，12px，输入框下方 4px
```

**表单顶部汇总（辅助，适用于复杂表单）**
```
+------------------------------------------+
|  ⚠ 以下字段需要补充：                   |
|  · 请输入学生姓名                         |
|  · 请选择辅导科目                         |
+------------------------------------------+
```

### 2.3 各字段验证规则

| 字段类型     | 验证规则                                      | 错误提示文案                         |
|--------------|-----------------------------------------------|--------------------------------------|
| 必填文本     | 非空、trim 后非空                             | 请输入 [字段名]                      |
| 手机号       | 11位数字，1开头                               | 请输入正确的手机号码                 |
| 日期         | 合法日期格式                                  | 请选择有效日期                       |
| 时间段       | 结束时间 > 开始时间                           | 结束时间必须晚于开始时间             |
| 金额         | 正数，最多2位小数                             | 请输入有效金额                       |
| 文件大小     | 单文件 ≤ 50MB                                | 文件大小不能超过 50MB                |
| 课程冲突     | 异步校验（与已有课程时间不重叠）              | 该时间段已有 [学生名] 的 [科目] 课，请调整|

### 2.4 成功反馈规范

**Toast 成功提示（轻量操作）**
```
位置：屏幕顶部居中（Web）或屏幕中央（小程序系统 Toast）
样式：绿色背景，白色文字，带勾选图标，2秒后自动消失
文案规范：
  - 保存成功
  - 学生信息已更新
  - 作业布置成功，已通知 3 名学生
  - 收款记录已保存
```

**内联成功状态（表单场景）**
```
提交按钮变为绿色 + 勾选图标，持续 1.5s 后恢复原状或跳转
适用于：快速编辑场景，不需要跳转页面的操作
```

## 三、弹窗 / 抽屉使用规范

### 3.1 使用场景对照

| 组件类型       | 适用场景                                         | 不适用场景                         |
|----------------|--------------------------------------------------|------------------------------------|
| Modal 弹窗     | 确认操作（删除等危险操作）、简单表单（字段 ≤ 6 个）、信息展示 | 复杂表单（字段多、需要上下文参考）|
| 抽屉（Drawer） | 详情查看、关联内容编辑（需要同时参考列表）       | 需要独立专注填写的表单            |
| 独立页面/路由  | 复杂表单（字段 > 6 个）、富文本编辑、多步骤操作  | 简单确认、快速预览                |

### 3.2 Modal 弹窗规范

**尺寸标准（Web 端）**
```
小型（确认弹窗）：width 400px
中型（简单表单）：width 560px
大型（复杂内容）：width 680px - 800px
超大型（富文本）：width 900px 或全屏
```

**小程序端 Modal**
使用微信原生 `wx.showModal()`（危险确认）或自定义 modal 组件（表单）。

**结构规范（Web）**
```
+--------------------------------------------+
| 标题（16px, Bold）              [X 关闭]  |  <- 固定顶部，不随内容滚动
| ————————————————————————————————————————   |  <- 分隔线
|                                            |
|  内容区域（可滚动，最大高度 70vh）          |
|                                            |
| ————————————————————————————————————————   |  <- 分隔线
| [次要操作] （左对齐，如"删除"）  [取消] [主操作] |  <- 固定底部
+--------------------------------------------+
```

**行为规范**
- 点击弹窗外部蒙层：关闭弹窗（除危险操作确认弹窗外）
- 按 Esc 键：关闭弹窗
- 弹窗打开时：背景页面禁止滚动
- 弹窗内容超出高度：内容区域独立滚动

**危险操作确认弹窗规范**
```
+--------------------------------------------+
| 确认删除                            [X]   |
| ————————————————————————————————————————   |
|                                            |
|  确认删除学生"张三"？                      |
|  删除后其课程、作业等历史记录将保留，        |
|  但学生基本信息无法恢复。                   |
|                                            |
| ————————————————————————————————————————   |
|                     [取消]  [确认删除]      |
|                           (红色危险按钮)   |
+--------------------------------------------+
```

规则：
- 说明删除的具体对象（带引号标注名称）
- 说明不可恢复性
- 危险按钮在右侧（符合用户习惯，但需要用户主动点击）
- 点击弹窗外不关闭（防止误关后误以为操作成功）

### 3.3 抽屉（Drawer）规范

**尺寸（Web 端）**
```
小型：width 400px  - 快速预览
中型：width 540px  - 详情查看（带 Tab 切换）
大型：width 640px  - 包含编辑表单的详情
```

**行为规范**
- 从右侧滑入，动画 250ms ease-out
- 点击左侧蒙层关闭
- 关闭前如有未保存修改：提示"有未保存的修改，确认关闭？"
- 抽屉内部可独立滚动
- 多个抽屉叠加（如详情内打开编辑）：第二个抽屉再往右偏移 320px

## 四、移动端手势支持（小程序端）

### 4.1 支持的手势操作

| 手势             | 触发位置                   | 执行操作                         |
|------------------|----------------------------|----------------------------------|
| 下拉刷新         | 页面内容区域顶部下拉        | 刷新当前页面数据                 |
| 左右滑动         | 课程日历月份区域            | 切换上一月 / 下一月              |
| 左滑列表项       | 资料列表、作业列表          | 出现"删除"操作按钮（仅老师端）  |
| 长按             | 资料文件项                  | 弹出操作菜单（下载/预览/分享）  |
| 双指缩放         | 图片预览                    | 放大/缩小图片（系统默认支持）   |

### 4.2 下拉刷新规范
```
触发条件：页面滚动到顶部后继续下拉 50rpx
视觉反馈：顶部出现系统默认刷新指示器（微信小程序 enablePullDownRefresh）
完成后：指示器消失，列表更新，顶部显示"刚刚更新"小提示（可选）
```

### 4.3 左滑操作规范（列表项）
```
触发条件：在列表项上左滑 > 60rpx
弹出操作：
  资料列表：[下载] [删除]（仅在老师管理端小程序中实现，学生端不支持）
结构：
  +----------------------------------+----+----+
  | 文件名          内容...           |下载|删除|
  +----------------------------------+----+----+
  下载按钮：蓝色背景 80rpx
  删除按钮：红色背景 80rpx

取消：右滑复位，或点击其他区域
注意：危险操作（删除）需二次确认（wx.showModal）
```

### 4.4 小程序页面跳转规则

| 跳转类型       | 使用场景                         | API 调用                 |
|----------------|----------------------------------|--------------------------|
| 保留当前页跳转 | 进入详情页（可返回）             | `wx.navigateTo`          |
| 切换 Tab 页    | TabBar 点击                      | `wx.switchTab`           |
| 关闭当前跳转   | 登录成功后跳转首页               | `wx.redirectTo`          |
| 返回           | 详情页返回列表                   | `wx.navigateBack`        |

## 五、响应式断点规范（Web 端）

### 5.1 断点定义

| 断点标识 | 最小宽度  | 典型设备                        |
|----------|-----------|---------------------------------|
| xs       | 0px       | 手机竖屏（兜底）                |
| sm       | 576px     | 手机横屏 / 小平板               |
| md       | 768px     | 平板竖屏（iPad）                |
| lg       | 992px     | 平板横屏 / 小屏笔记本           |
| xl       | 1200px    | 标准笔记本（1280px）            |
| xxl      | 1600px    | 大屏显示器                      |

**目标优先级**：老师主要在笔记本/台式机上使用，设计以 lg（992px+）为主。sm 以下做基本可用性保障即可。

### 5.2 各断点布局变化

#### 侧边栏

| 断点  | 侧边栏状态                              |
|-------|-----------------------------------------|
| xs    | 完全隐藏，通过汉堡菜单呼出（抽屉形式）  |
| sm    | 隐藏，汉堡菜单呼出                      |
| md    | 图标模式（宽 64px，hover 展开 tooltip） |
| lg+   | 完整展示（宽 220px）                    |

#### 统计卡片行（仪表盘）

| 断点  | 每行卡片数 | 栅格配置 |
|-------|------------|----------|
| xs    | 1 个       | span 24  |
| sm    | 2 个       | span 12  |
| md    | 2 个       | span 12  |
| lg+   | 4 个       | span 6   |

#### 表格

| 断点  | 表格变化                                |
|-------|-----------------------------------------|
| xs-sm | 隐藏次要列（时长、备注等），只保留关键列 |
| md    | 隐藏部分列，操作列改为图标按钮          |
| lg+   | 完整显示                                |

#### 弹窗（Modal）

| 断点  | Modal 宽度变化                          |
|-------|-----------------------------------------|
| xs-sm | 全屏（100vw x 100vh），从底部弹出       |
| md+   | 固定宽度（400px / 560px / 680px）       |

### 5.3 触摸设备适配（平板/手机使用 Web 端）

```
当检测到 touch 设备时（@media (pointer: coarse)）：
- 所有可点击元素最小触摸区域：44px x 44px
- 表格行高增加到 60px（方便手指点击）
- 取消 hover 效果（避免 iOS 点击后残留 hover 状态）
- 侧边栏默认收起，通过手势或按钮展开
```

## 六、通知与提示规范

### 6.1 Toast / Message 规范

**使用场景与样式对照**

| 类型    | 颜色      | 图标          | 持续时间 | 触发场景                    |
|---------|-----------|---------------|----------|-----------------------------|
| 成功    | 绿色系    | CheckCircle   | 2s       | 保存成功、删除成功、上传完成 |
| 警告    | 橙色系    | Warning       | 3s       | 提示注意、操作有副作用       |
| 错误    | 红色系    | CloseCircle   | 4s       | 操作失败、网络错误           |
| 信息    | 蓝色系    | InfoCircle    | 2s       | 一般提示、状态更新           |

**位置**
- Web 端：页面顶部居中，固定定位，多个时向下堆叠（最多3条）
- 小程序端：使用系统 `wx.showToast()`（成功/加载中），复杂提示用自定义组件

**文案长度限制**
- Web Toast：不超过 40 个字符（单行）
- 小程序 Toast：不超过 7 个字（系统限制）

### 6.2 气泡确认（Popconfirm）

适用于：表格行内的快速确认操作（比 Modal 更轻量）。

```
触发：点击"删除"按钮
显示：
  +-------------------+
  | 确认删除？         |
  | [取消] [确认]      |
  +-------------------+
  （小气泡，出现在按钮上方）

适用场景：表格行删除操作（数据量小、影响范围小）
不适用：删除学生（影响大，用 Modal 二次确认）
```

### 6.3 全局通知（Notification）

适用于：系统级别的通知推送（非用户操作触发）。

```
位置：右上角，固定定位
宽度：360px
最多同时显示 3 条，超出时最旧的自动关闭

结构：
+------------------------------------------+
| [图标] 通知标题                  [X关闭] |
|        通知详情描述（最多2行）            |
|        2分钟前                   [查看]  |
+------------------------------------------+
```

适用场景：
- 新作业提交提醒
- 课程开始前提醒（页面开着的情况下）

## 七、数据更新策略

### 7.1 乐观更新（Optimistic Update）

适用操作：勾选完成、标记状态等低风险、可撤销操作。

```
流程：
1. 用户点击操作
2. 立即更新 UI（不等待接口）
3. 异步发起接口请求
4. 成功：显示成功 Toast
5. 失败：回滚 UI 到操作前状态 + 显示错误提示
```

### 7.2 等待确认更新（Wait for API）

适用操作：删除、新增、提交收款等高风险操作。

```
流程：
1. 用户点击操作
2. 按钮进入加载态
3. 等待接口返回
4. 成功：更新 UI + 显示成功 Toast
5. 失败：保持当前 UI 不变 + 显示错误提示
```

### 7.3 自动刷新

```
以下场景触发自动数据刷新：
- 从编辑弹窗/抽屉关闭后：刷新对应列表
- 从新增 Modal 关闭后：刷新对应列表（滚动到顶部）
- 小程序从后台切回前台（App.onShow）：刷新当前 Tab 数据
```
