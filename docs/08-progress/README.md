# 进度同步中心

> 状态：当前
> 范围：全项目
> 更新：2026-04-28

本目录用于存放正在推进中的计划进度、负责人同步和多人并行工作流状态。它回答“现在谁在做什么、做到哪里、卡在哪里、下一次同步需要交付什么”。

## 定位

`docs/04-implementation/` 负责说明实现方案和任务拆解，`docs/08-progress/` 负责记录当前进行态，`docs/07-changes/` 负责沉淀已确认或已完成的重要结果。

不要把长期方案、PRD、接口设计或完成态变更只写在进度文档里。进度文档中的结论一旦变成事实，应同步回产品、设计、架构、实现、测试或变更记录。

## 状态值

| 状态 | 含义 |
| --- | --- |
| 待开始 | 已识别但尚未进入开发或整理 |
| 进行中 | 有人正在推进，需持续同步 |
| 阻塞 | 需要外部信息、依赖或决策解除阻塞 |
| 待验证 | 实现或文档已完成，等待测试或验收 |
| 已完成 | 已验证并完成交接，可沉淀到变更记录 |
| 暂停 | 当前不推进，但保留后续可能 |

## 文件组织

- `docs/08-progress/project-status.md`：项目级进度总览，只保留当前进行态和下一步。
- `docs/08-progress/workstreams/`：多人并行任务流，每个工作流单独一份文档，避免多人同时编辑一个大计划文件。

## 更新规则

- 开始一个超过半天或跨模块的任务时，新增或更新对应工作流文档。
- 负责人、状态、风险、下一步、同步输出变化时，更新 `project-status.md` 和相关工作流文档。
- 工作流完成后，将完成态事实沉淀到 `docs/07-changes/`；如果产品、设计、实现或测试事实变化，也同步更新对应目录。
- 工作流结束超过一轮迭代且无继续价值时，可以移入 `docs/99-archive/`，并同步更新 `docs/04-implementation/doc-map.md`。

## 多人并行约定

- 每个并行工作流有独立文档和明确负责人。
- 多个研发或 agent 不应长期同时编辑同一个工作流文档；如确需协作，应在文档中拆分子负责人和交接点。
- 进度同步只记录必要上下文，不粘贴完整 AI 对话。
- 每次同步至少更新：当前状态、最近输出、阻塞点、下一步。
